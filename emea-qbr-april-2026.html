Data Architecture

Complete vs Composable Mode

Complete Mode (Traditional)

Data ingested and unified within TD. Parent Segment + child segments + decisioning prior to activation.

Sources→Source DB→PS Data DB→Audience DB→Activation

All processing within Treasure Data workflows.

Composable Mode

CDW is source-of-truth. TD serves as virtual governance and decisioning/intelligence layer.

Sources→CDW→Audience Studio→Activation

Data pipeline in CDW. TD provides governance + intelligence.

Key Setup Differences
  • Import all data into CDW from all source systems
  • Build Audience DB tables (Customers, Behaviors) in CDW
  • Setup Composable PS Config with CDW connection

Composable FAQ

When episodic data lives outside DBX/SNOW, what's the recommendation?

Approach 1: TD as Transfer Layer

Use Treasure Data as an interim data transfer layer to move data from source into the CDW.

Data Connector→ Import→ Workflow→ Result Export to CDW

Best when the primary requirement is simply to transfer data from a source to CDWs.

Approach 2: Iceberg Table Support

If the customer is comfortable with the dataset residing within TD but still needs to expose it to a CDW, implement Iceberg table support.

This allows the dataset to be incorporated as a data source for "golden records" within the CDW environment.

Best when data can live in TD but needs to be queryable from the CDW.

Composable FAQ

Where should IDU (Identity Unification) live?

Treasure Data can own the IDU workflow process and run queries on CDWs to establish unified profile IDs — while keeping data and query resources within the CDW environment at all times.

  • TD manages the IDU workflow
  • Queries run directly on CDWs
  • Unified profile IDs are established in-place
  • Data and compute stay within the customer's CDW
IDU Skill — Current Status & Next Steps

The APS team has an existing IDU Skill that converts unify.yml to ID unification workflows on Snowflake/Databricks.

  • Current capability: APS IDU skill generates SQL-based unification workflows for Snow/DBX from a unify.yml config
  • Output files: snowflake_idu.dig, config.yml, snowflake_sql/
  • Next step: Build a skill to generate unify.yml interactively via Treasure Work (in progress, Yasu)
  • Goal: End-to-end IDU setup through conversational interaction in Treasure Work — demo to follow

What if an existing customer wants to go composable?

Yes, migration would be needed.

Composable FAQ

How will RT use cases work with Composable?

Same as Complete mode — except the batch data stream is composable.

  • Real-time and batch processes unified through integration of RT system and Composable Audience Studio
  • RT ID stitching executes within the Reactor Engine, consistent with RT2 Complete mode

How GTM Influences Product

How the field can better work with Product on deals, and influence the roadmap via feature enhancement requests

1Log Product Gaps in HubSpot

The primary mechanism for field influence — replaced Vivun earlier in 2026.

  • Any AD, CSM, or SE can submit a product gap
  • Tie each gap to a Company (account gap) or Deal (deal-specific)
  • Always attach a dollar value / ARR — gaps without revenue carry far less weight
2Product Gap Council (Monthly)

GTM leaders surface the field's most impactful gaps directly to PMs.

  • Each regional lead (NALA: David Choi, EMEA: Charles Vatin, Post-Sales: Scott Mitchell, JP: Kohei Yamamori) brings top 5 gaps

  • PMs respond with prioritisation status and timelines
  • Accepted gaps become PRP tickets in Jira Polaris
3Product Office Hours (Bi-Weekly)

Every other Thursday, 9:00–10:00 AM PST — open forum for timelines, roadmap items, and releases.

  • Roadmap updates, prototypes, demos, and open Q&A
  • Official "when is X shipping?" channel — PMs won't join individual SBR/account meetings
4Escalation for Deal-Critical Gaps

For a feature gap actively blocking or threatening a deal:

  • Raise with your GTM leader immediately
  • They escalate directly to Kristen Zhou (CPO)
  • Don't use the monthly council for time-sensitive deal escalations
5Top 50 Accounts — Named PM

Strategic accounts have a dedicated PM assigned. Account team should still log gaps in HubSpot, but can co-review gap status periodically with the assigned PM directly.

6On-site Support

At the request of a GTM Team, PMs can be dispatched to a prospect or customer for on-site demos, walk-throughs and presentations. Reach out to Kristen for Product Approval.
