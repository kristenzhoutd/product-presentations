Data Architecture

Complete vs Composable Mode

Complete Mode (Traditional)

Data ingested and unified within TD. Parent Segment + child segments + decisioning prior to activation.

Sources→Source DB→PS Data DB→Audience DB→Activation

All processing within Treasure Data workflows.

Composable Mode

CDW is source-of-truth. TD serves as virtual governance and decisioning/intelligence layer.

Sources→CDW→Audience Studio→Activation

Data pipeline in CDW. TD provides governance + intelligence.

Key Setup Differences
  • Import all data into CDW from all source systems
  • Build Audience DB tables (Customers, Behaviors) in CDW
  • Setup Composable PS Config with CDW connection

FAQ & Q&A
Common questions from the field
Questions

Frequently Asked Questions

When episodic data lives outside DBX/SNOW, what's the recommendation?
Use TD as interim data transfer layer (Data Connector → import → workflow → result export to CDW). Alternatively, implement Iceberg table support for 'golden records' within the CDW.
Where should IDU (Identity Unification) live?
TD can own the IDU workflow and run queries on CDWs to establish unified profile IDs while keeping data and query resources within the CDW environment.
What if an existing customer wants to go composable?
Migration is needed. Customer/Partner should own it (CDW is their asset). PS team can do it at premium cost.
How will RT use cases work with Composable?
Same as Complete mode — except batch data stream is composable. RT ID stitching executes within the Reactor Engine, consistent with RT2 Complete mode.
What is Agentic Audience Studio?
AI-Native Audience Studio is being planned on the new Agent Architecture. Full enablement scheduled for 4/16.
When a capability is delayed, how are GTM teams informed?
Product Org publishes a diff and covers it in Product Office Hours. GTM members should attend or review recordings. Questions on #product.
