Security

Enterprise Security Architecture

Purpose-built for regulated industries and zero-trust environments

Authentication & Identity

OAuth 2.0 with PKCE flow — tokens never exposed in URLs. Supports SAML, OIDC, and Native TD credentials. Encrypted token storage on Desktop and Mobile. Automatic session refresh before expiration.

Server-Side Sandbox

All AI code execution happens server-side in a restricted sandbox. No filesystem access. No arbitrary network access. Credentials managed server-side, never stored in browsers. This is the architecture security teams want to see.

Network Audit & Monitoring

Full Network Audit Log with temporal filtering (1h to 30d), domain/IP/port filtering, and real-time statistics. IP Allowlist with fixed static outbound IPs. See exactly what the AI accessed, when, and where.

Policy-Based Access Control

Policy-based permissions inherited from Treasure Data Console. All queries, segments, and API calls execute through the user's own credentials. Same governance model your security team already approved for TD.

Key Message for IT Buyers

Treasure AI Studio inherits the security posture of the Treasure Data platform. If they've approved TD, there's no new vendor risk assessment needed. Offer a joint security review call with their CISO to walk through the architecture.

Market Positioning
What makes the Treasure AI approach different
Technology Edge

The Treasure AI Approach

Built on the Anthropic Agent framework — the same architecture powering the most advanced AI products in the market. We ship what others announce.

Built on Claude Agent SDK

Not a wrapper around an API. Treasure AI Studio is built directly on Anthropic's Claude Agent SDK — the same framework used by the most advanced AI applications in the world. This gives us native access to the latest model capabilities the moment they ship.

Architected for Extensibility

Studio's agent architecture is designed to support open standards like Model Context Protocol (MCP) for AI-to-tool communication. Third-party integration capabilities are on the FYQ1 roadmap — when they ship, the foundation is already in place. No retrofit required.

Speed to Innovate

When Anthropic releases a new model, we adopt it in days — not quarters. Claude Opus 4.6, Sonnet 4.6, and Haiku 4.5 are all available at GA. Competitors are still running last-generation models while we ship the frontier.

Agent Architecture, Not Prompt Chaining

Our 5-stage Agent Orchestrator (Context → Decompose → Tools → Execute → Assemble) is a true agentic system. It plans, selects tools, executes in a server-side sandbox, and assembles results — not a chain of prompts pretending to be intelligent.

Skills: Modular Intelligence That Compounds

30 pre-built skills ship at GA — and customers can build custom skills in natural language to meet their unique workflows and use cases. Each new skill makes the entire platform smarter. Competitors embed AI helpers; we ship an extensible intelligence layer.

Three Model Tiers, One Click

Users switch between Fast (Haiku 4.5), Balanced (Sonnet 4.6), and Pro (Opus 4.6) mid-conversation. Match AI power to the task — quick lookups don't need frontier reasoning, deep analysis doesn't need to wait for a cheaper model.

Why This Matters
